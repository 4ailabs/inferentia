# math-engine

Python math engine for Inferentia. Runs in Vercel Sandbox (isolated Firecracker microVM).

## Components

- **`imprint_gmm.py`** — Bayesian Gaussian Mixture Model over 10-dim physiological space. Classifies patient phenotype → posterior over imprints.
- **`active_inference.py`** — POMDP via `pymdp`. Infers active predictive priors from multimodal evidence.
- **`snp_bayes.py`** — Bayesian SNP posterior via NumPyro. Conditioned on imprint + phenotype.
- **`signature_predictor.py`** — DBN from imprint → expected metabolic signature with credible intervals.
- **`parse_signatures.py`** — Parses `docs/imprint_signatures/*.md` into numeric GMM parameters.

## Dependencies

## How it's called

From Next.js API route → Vercel Sandbox spawns Python process:

```ts
const result = await sandbox.run("python -m math_engine.infer", {
  input: JSON.stringify({ phenotype, evidence }),
});
```

## Design principles

1. **LLM delegates math, does not approximate it.** All posteriors are computed numerically, not generated by the LLM.
2. **Uncertainty quantified at every step.** No point estimates without credible intervals.
3. **Imprint tables are the source of truth.** Priors come from `docs/imprint_signatures/`, not hardcoded.
4. **Reproducible.** Same input → same output. No hidden randomness.
